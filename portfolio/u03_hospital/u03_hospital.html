Introduction

This product is a hospital registration system that we create in the Prototyping Innovation and Verification class. The purpose of the system is that we hope to reduce the burden on the medical staff. In the course, we almost built one prototype a week to get feedback and improve it in the next week.

Process

Prototype01_Relief Balcony

In the beginning, in order to deal with the problem that patients will keep knocking the door and it will bother the nurses and make them have too much work to do, we came up with an idea about building a Relief Balcony for patients to chat with each other, enjoy the plants and also can get the number from the clinic. To demonstate the idea, we take the video below to show it.

And then, we got some feedback after demonstrating our ideas.

  1. If there are crowded, how can we deal with the situation?
  2. How much water are we going to for patients to use?
  3. Maybe we should think more about how patients interact with each other
  4. The screen of the clinic number are too small for patients to relieve

We also think about some solutions for the above feedback.

  1. If there are crowded, maybe we can have more balcony or relief places for people.
  2. In order not to let plants drown, there will be just moderate water for patients to use.
  3. We think we can have more themes, like games, nature, or interaction application for people to interact with each other.
  4. To deal with the screen, we can make the screen bigger and also use some voice or vibration hint for patients to get the information from clinic.

Prototype02_Registration System Flow

For the further process, we began to plan the flow of the hospital registration system.



We think that the verify system will be very important part of the system, so we plan to let the system have two verification stages. Patients needed to choose ID card or medical insurance card for first stage verification. The second stage is that patients needed to pass face recognition or fingerprint recognition for hospital registration.



Prototype03_Build Kiosk Prototype

The next step of the prototype is that we began to plan the first user interface of the hospital registration system. Besides, we also built a real Kiosk prototype to simulate the user context, like the screen size, and height, etc.


And then, we got some feedback after demonstrating our ideas.

  1. The background image overlaps with the previous illustration, causing recognition problems
  2. Text liner is not easily identifiable
  3. There is a misunderstanding problem with the text.

We also think about some solutions for the above feedback.

  1. Changing transparency to be non-disruptive.
  2. Changing the text to bold type, and then refer to the beautiful signboard.
  3. Modifying the text to fix the misunderstanding problem.

Prototype04_Registration System Revised

Then we make the revised version of the above user interface.



And then, we got some feedback after demonstrating our ideas.

  1. Font style is not very suitable.
  2. The size of the button need to be reconsidered.
  3. The sign of X maybe is not very good to present in the interface
  4. The map should have the function can zoom in/zoom out



Prototype05_Registration System Revised

To deal with above problems, we make the next revised version.

And then, we got some feedback again.

  1. The process on the right side are too like buttons
  2. Log out and help buttons don't match the colors of other buttons
  3. Help button should have fool-proof design
  4. The second verification method should set face recognition as default
  5. If the clock icon didn't have the function should take it off
  6. After clicking the help button should not let patient wait in front of the Kiosk and occupy the machine

Prototype06_Registration System Revised_Coding

To deal with above problems, we make the next revised version.

And then we wrote down a user test script for user testing. In the user testing, we planned to let users test two missions. The first task is using medical insurance card to register. The second task is using ID card to register.

User Testing Script(Chinese)

We got lots of feedback from users and all the comment help us build the final result.

Reflection

After this class, I have experienced what is the rapid prototyping process and agile development. Although the revised iterations are very tense, I really learned a lots in this course. Not only the practical tools to build the interface of the Kiosk, but also how to test the users to get appropriate feedback for us to do the further improvement. This course really help me a lot to understand what is the real procedure in the enterprise.
